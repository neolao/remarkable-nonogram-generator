# Module: web-server
**Role:** Fastify HTTP server exposing the API and serving the static frontend. Exposes the reMarkable connection and send endpoints, a version check, the full `/api/nonograms/*` CRUD route set (list, create, fetch, update, delete), and stateless SVG preview / PDF generation endpoints; the frontend editor UI is not built yet.
**Files:** `packages/web/src/server.ts`, `packages/web/src/nonogram-routes.ts`, `packages/web/public/index.html`, `packages/web/public/app.js`, `packages/web/public/style.css`
**Exports:**
- `buildServer(options?): FastifyInstance` — registers static file serving, `GET /api/version`, the reMarkable routes ([[remarkable-cloud]], now also given the nonogram store so it can serve `POST /api/nonograms/:id/send`), and the nonogram routes; `options.nonogramsPath` overrides the default storage directory ([[nonogram-persistence]])
- `DEFAULT_PORT`, `resolvePort(env?)` — the server listens on `PORT` if set (validated as an integer in `1..65535`), else `DEFAULT_PORT`
- `registerNonogramRoutes(app, store)` — wires `GET /api/nonograms` to `NonogramStore.list()` (lightweight summaries, no cell data); `POST /api/nonograms` validates the grid via `createNonogram` and creates it (201) or returns 400 with a clear error; `POST /api/nonograms/preview` validates a grid the same way and returns its rendered SVG (200, `image/svg+xml`) via `renderNonogramToSvg` ([[nonogram-rendering]]) without touching storage, or 400 with a clear error; `POST /api/nonograms/generate` validates a grid the same way and returns its rendered PDF (200, `application/pdf`) via `renderNonogramToPdf` ([[nonogram-rendering]]) without touching storage, or 400 with a clear error; `GET /api/nonograms/:id` returns the full saved grid (200) or 404; `PUT /api/nonograms/:id` validates and updates an existing nonogram (200) or 404 if the id is unknown; `DELETE /api/nonograms/:id` removes it (204) or returns 404 if the id is unknown
**Depends on:** `modules/remarkable-cloud.md`, `modules/nonogram-persistence.md`, `modules/nonogram-domain.md` (for `createNonogram` validation), `modules/nonogram-rendering.md` (for `renderNonogramToSvg` and `renderNonogramToPdf`), `@remarkable-nonogram-generator/core` (for `CORE_VERSION`)
