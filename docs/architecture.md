# Architecture

This is a two-package Node.js/TypeScript monorepo: a `core` library holding
all the puzzle and reMarkable-integration logic, and a `web` package that
exposes it through a Fastify API and a plain HTML/JS/CSS frontend. `core` has
no dependency on the browser or on any web framework, so the puzzle logic can
be tested and reasoned about entirely on its own.

## Core: puzzle logic and reMarkable integration

`core` is organized in three layers:

- **Domain** — the pure puzzle model: a nonogram's grid of filled/empty
  cells, the row and column clues derived from it, and the logic solver that
  reconstructs a grid from clues alone (used when importing a puzzle from an
  external source, so the source's own cell data is never trusted directly).
- **Application** — the use cases built on top of the domain model: sending a
  saved puzzle to a reMarkable tablet, and importing a puzzle from a
  nonograms.org page URL.
- **Infrastructure** — the technical building blocks those use cases rely on:
  rendering a grid to SVG (live preview) and to a print-ready PDF sized for
  the reMarkable 2, the reMarkable Cloud client (pairing and upload), reading
  a nonograms.org page through a headless browser, and the JSON/zip formats
  used to back up or move puzzles.

## Web: API and frontend

`packages/web` is a Fastify server that exposes this logic over HTTP — saved
puzzle CRUD, SVG preview and PDF generation, sending to reMarkable, importing
from a URL or from a file, and bulk export/import of the whole puzzle list —
plus a static, build-free frontend (plain HTML pages with native JS modules)
covering the home page, the manual grid editor, and the import page. Saved
puzzles and the reMarkable pairing credentials are stored as files under the
user's home directory, one file per puzzle.

## Deployment

The app can also run as a single Docker container; see
[docs/docker.md](docker.md) for the CLI and Synology NAS instructions.
